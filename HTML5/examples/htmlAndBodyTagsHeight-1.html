There was something to do with the heights of the body and the HTML root element that I have now forgotten. I had experimented quite a bit with it when I was revising HTML recently about a year and a half ago. It was that they were somehow independent of each other and there was some quirky surprising behavior. I think I even made a stackoverflow question about it expressing my surprise.

So, here I am, testing that same thing again just so as to remember it, so I can apply it in redesigning some pages that I am doing now.

Ok, running just this much, two things I note right off the bat that I'd noted earlier:

  1. The body element won't stretch to fit the full height of the HTML root element, by default. You have to make it do that.
  2. There's an automatic margin that the body element has. I didn't set it, but observe the margin around the body on all four sides. I am not sure if it is the margin of the body element or the padding of the root HTML element. I'll have to find out by setting each of these to zero one-by-one and see. That's what I am going to do next.
  3. Setting the height to 100% on the body element still does not stretch it to the full height of the root element. In fact, it has no effect on the body element at all. I have got to find that Stack Overflow question of mine as to what my surprise was. It was something else, though.
  4. Setting margin: 0px on the body element removes its left and right margins but does not remove the margin on the top and also has no effect on its height. By default, the body only takes up as much height as is required to fit its contents.
  5. Ok. here's the thread. I found it. Is the body element's height not 100% of its container by default in the static positioning normal layout? (Permalink: https://stackoverflow.com/q/50240214/303685)
  6. The take-ways from the SO thread are listed as the following bullet points.
  7. That's right. This is a quirk. When left to the default positioning: static, the body element's height is only as much as it needs to fit its entire contents. It does not stretch to the height of the HTML element.
  8. We can make it stretech by either changing its positioning to absolute and settings it height to 100% and left, and other dimensions as we like. Or, we could leave the positioning to be static but change the height to auto. That would make it cover its height to 100% of the viewport or the HTML root element.
  9. If you do not set the background-color of the HTML root element, it inherits the background-color of the body element making it look like the body element is covering the entire height of the viewport, which is not the case.
